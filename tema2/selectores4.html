Pseudoselectores de formularios

Existe una serie de pseudoclases en CSS que pueden ser utilizadas para usar en formularios de una página web. Estas pseudoclases permiten seleccionar elementos para darle estilo dependiendo de temas relacionados con los formularios o los campos que están en el interior de un formulario.

- Los vamos a dividir en 3 clases:

  • Pseudoclases de interacción en formularios: Seleccionar elementos cuando cambia el estado de un elemento.
  • Pseudoclases de estado en formularios: Seleccionar elementos cuando se encuentran en un estado concreto.
  • Pseudoclases de validación: Seleccionar elementos si cumplen o no un cierto criterio de validación.

Pseudoclases de interacción en formularios

- :checked para seleccionar aquellos input de tipo radio o checkbox que estén marcado/s.

- :indeterminate Selecciona el elemento cuando la casilla está en un estado indeterminado.

:checked

La pseudoclase :checked en CSS se utiliza para aplicar estilos a elementos de formulario (como checkboxes y radiobuttons) cuando están seleccionados o marcados por el usuario. En otras palabras, se aplica cuando el atributo checked del elemento es verdadero.

Los input de tipo checkbox o radiobutton son nativos del navegador, es decir son renderizados por el propio motor del navegador a los cuales no se les van a aplicar todas las reglas css que se le aplican a otros elementos, por ejemplo el color de fondo no se va a poder de forma normal. Ya veremos como cambiar los checkbox, radiobutton y select más adelante.

Sí nos va a permitir cambiar los label o el span de los checkbox o radiobutton que esten marcados y aplicarle algún estilo.

El label de los checkbox que estén activos los pones la fuente azul.

Otros usos:

La pseudoclase :checked también puede usarse con elementos radio y con las opciones dentro de un elemento <select>. Por ejemplo, podrías cambiar el estilo de una opción seleccionada en un menú desplegable: 

                        option:checked {
                            font-weight: bold;
                            color: green;
                        }
Este código haría que la opción seleccionada en un menú desplegable aparezca en negrita y de color verde.

:indeterminate

La pseudoclase :indeterminate en CSS se utiliza para seleccionar elementos de formulario (como input[type="checkbox"], input[type="radio"], y progress) cuyo estado es indeterminado. Esto ocurre cuando un checkbox o radio button no está ni checked ni unchecked, o cuando un elemento <progress> no tiene un valor definido.

Ejemplos:

  1. Checkbox indeterminado: Un input[type="checkbox"] cuyo atributo indeterminate se establece a true mediante JavaScript. Un input type="checkbox" puede tener su propiedad indeterminate establecida a true mediante JavaScript. Esto puede ocurrir cuando un checkbox representa un estado "parcialmente seleccionado" en un grupo de elementos.
  2. Radio buttons indeterminados: Varios input[type="radio"] con el mismo atributo name pero ninguno con el atributo checked. Si tienes un grupo de radio buttons con el mismo nombre, y ninguno de ellos tiene el atributo checked establecido, todos pueden ser considerados indeterminados.
  3. Elemento progress indeterminado: Un elemento <progress> sin un valor definido, mostrando una animación de carga continua. Una barra de progreso que no tiene un valor definido (o su valor es negativo) se considera indeterminada y puede ser estilizada con :indeterminate. ()

Explicación:

  • El primer selector aplica un fondo amarillo a cualquier checkbox que tenga su propiedad indeterminate establecida a true.
  • El segundo selector aplica un fondo rojo a cualquier elemento <progress> que sea indeterminado, es decir, que no tenga un valor definido o que su valor sea negativo.

En resumen, la pseudoclase :indeterminate es útil para aplicar estilos específicos a elementos de formulario que se encuentran en un estado indeterminado, lo que puede ayudar a comunicar visualmente este estado especial al usuario.

Etiqueta <progress>

La etiqueta <progress> en HTML se usa para mostrar el progreso de una tarea. Visualmente, se presenta como una barra de progreso, ya sea horizontal o vertical, y su contenido dependerá de cómo se defina en el código HTML. Básicamente, indica el avance de una tarea, mostrando un porcentaje o un valor específico, y su apariencia puede variar según el navegador.

Estructura básica: 

                        <progress value="50" max="100"></progress>

  • value: Este atributo indica el valor actual del progreso. En este ejemplo, es 50.
  • max: Este atributo indica el valor máximo del progreso. Aquí, es 100.

Si el atributo value es igual a 50 y el max es 100, la barra mostrará el 50% de progreso.

Otras formas de usarla:

Sin atributos value y max: Si no se especifica value y max, se mostrará un indicador de progreso indefinido (una barra animada sin valor específico).

JavaScript: Puedes usar JavaScript para actualizar dinámicamente los valores de value y max según el progreso de una tarea.

En este ejemplo, la barra de progreso se actualiza cada segundo hasta llegar al 100%.

Consideraciones:

  • La etiqueta <progress> es un elemento de la especificación HTML5 y es compatible con la mayoría de los navegadores modernos.
  • La apariencia visual de la barra de progreso puede variar entre navegadores, ya que cada uno puede tener su propia implementación de estilos.
  • Para un mayor control sobre la apariencia, puedes utilizar CSS para personalizar el estilo de la barra de progreso.

Pseudoclases DE ESTADO en formularios

Existen varias pseudoclases para detectar si un campo de un formulario está activado o desactivado, o cierta información relacionada con su estado.

:enabled Utilizando la autoexplicativa pseudoclase :enabled, podemos seleccionar elementos que se encuentren activados (comportamiento por defecto)

:disabled Sin embargo, lo más interesante de este tema viene al poder darle estilo a elementos desactivados con la pseudoclase :disabled, donde se seleccionan los elementos a los que se le ha añadido el atributo disabled

:read-only La pseudoclase :read-only selecciona aquellos elementos input de un formulario que están marcados con el atributo de sólo lectura readonly. La diferencia entre un campo con atributo disabled y un campo con atributo readonly es que la información del campo con readonly se enviará a través del formulario, mientras que la del campo con disabled no se enviará.

:read-write Por otro lado, la pseudoclase :read-write es muy útil para dar estilos a todos aquellos elementos que son editables por el usuario, sean campos de texto input o textarea.

:placeholder-show Con la pseudoclase :placeholder-shown se nos permite seleccionar y dar estilo a los elementos que están actualmente mostrando un placeholder.

Pseudoclases DE VALIDACIÓN en formularios

En HTML5 es posible dotar de capacidades de validación a los campos de un formulario, pudiendo interactuar con ellos desde Javascript o incluso desde CSS. Con estas validaciones podemos asegurarnos de que el usuario escribe en un campo de un formulario el valor esperado.

:required Por lo general, los campos de un formulario son siempre opcionales. Para hacer obligatorio un campo, tenemos que indicar en el elemento HTML el atributo required, al cuál será posible darle estilo mediante la pseudoclase :required:

:optional Por otra parte, los campos opcionales son todos aquellos que no tienen el atributo required. Pueden seleccionarse con la pseudoclase :optional

:valid Actualmente, desde HTML5 se brinda un excelente soporte de validaciones desde el lado del cliente sin necesidad de Javascript 

<input type="text" name="age" pattern="[0-9]+" />

:invalid De forma opuesta, se pueden seleccionar elementos y aplicar ciertos estilos si no se cumple el patrón de validación 

<input type="text" name="age" pattern="[0-9]+" />

:in-range Lo ideal sería establecer un rango, algo que se suele hacer muy a menudo si tenemos campos numéricos de formulario mediante los atributos min y max 

<input type="number" name="age" min="18" max="100" />

:out-of-range Lo contrario de lo anterior 

<input type="number" name="age" min="18" max="100" />

Pseudoelementos

Los pseudoelementos son una característica de CSS que permite hacer referencias a «comportamientos virtuales no tangibles», dicho de otra forma, seleccionar y dar estilo a elementos que no existen en el HTML, o que no son un simple elemento en sí.

La sintaxis de los pseudoelementos está precedida de dos puntos dobles (::) para diferenciarlos de las pseudoclases, las cuales sólo tienen dos puntos (:).

::content La propiedad content admite parámetros de diverso tipo, incluso concatenando información mediante espacios. Podemos utilizar varios tipos de contenido

Ten en cuenta que la propiedad content sólo funciona dentro de los pseudoelementos ::before y ::after que explicaremos a continuación. No puede utilizarse en otro lugar.

::before / ::after Los pseudoelementos ::before y ::after permiten hacer referencia a «justo antes del elemento» y «justo después del elemento», respectivamente. Además de selectores básicos como clase o id, combinadores, atributos o pseudoclases, puedes añadir un pseudoelemento, precedido por doble :.

Pseudoelementos de resaltado

Existe una serie de pseudoelementos orientados a la selección o resaltado de texto en un documento HTML mostrado a través de un navegador.

::selection Cuando seleccionamos un fragmento de texto, el navegador suele aplicar un color de fondo que depende del sistema operativo, del tema, o similar.
